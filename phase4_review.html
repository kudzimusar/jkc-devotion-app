However if these Client components import supabase and run useEffect, and the Server page just wraps them, the metadata will export correctly but data fetching is still client-side. Multiple builds (v1–v4)
Note 5 separate build+commit cycles suggests the dynamic route for ministries caused repeated build failures. This is resolved but worth understanding — generateStaticParams must return all slugs at build time. If Supabase is queried inside generateStaticParams on a static export, it will fail if the DB is unreachable at build time. The hardcoded slug array approach in the prompt is safer.
